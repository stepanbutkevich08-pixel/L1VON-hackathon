# Фин.Универ x T1 - Команда L1VON
Кейс: оптимальное размещение 250 медицинских клиник по данным о 42 006 клиентах с учетом ограничений по вместимости и социальной доступности.

## Описание проекта
Python-скрипт рассчитывает координаты клиник так, чтобы минимизировать среднее время пути до ближайшей точки (метрика TTT) при лимите `1000` клиентов на клинику и с оценкой социального окружения (LCS). Основной пайплайн - `clinic_placement_solver.py`: инициализация k-means -> прогрев с учетом вместимости -> TTT-оптимизация -> локальный swap-поиск -> визуализация и метрики.

## Загрузка
Склонируйте репозиторий:
```bash
git clone https://github.com/stepanbutkevich08-pixel/L1VON-hackathon.git
cd L1VON-hackathon
```

## Запуск в локальной среде
Python 3.11+
```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
python3 clinic_placement_solver.py
```
В консоли будут TTT/CO/LCS, в `clinics.csv` - координаты клиник, в `plots/` - PNG с картой и гистограммой загрузки. Для быстрого вывода предрасчитанных центров можно запустить `python3 optimisation.py`. Для полного воспроизведения пайплайна с мягким уточнением - `python3 gen.py` (понадобится `torch`).

## Выходные данные
- `clinics.csv` - итоговые координаты размещенных клиник (x, y).
- `plots/01_map_clients_clinics.png` - точки клиентов и клиник.
- `plots/02_map_by_assignment.png` - клиенты, окрашенные по назначенной клинике.
- `plots/03_clinic_load_hist.png` - распределение загрузки клиник.
- TTT/CO/LCS печатаются в stdout после завершения.
